# 🏢 WorkPresso 내부 운영 및 개발 가이드라인 (v1.2)

이 문서는 WorkPresso 임직원들이 원활하게 협업하고 높은 수준의 제품을 개발하기 위한 표준 가이드라인입니다.

---

## 1. 💻 개발 컨벤션 및 배포 프로세스

### 1.1 Git 브랜치 전략 (Git Flow)
- 모든 개발은 `main` 브랜치에서 분기된 `feat/`, `fix/`, `docs/`, `refactor/` 브랜치에서 진행합니다.
- 브랜치 명명 규칙: `feat/이슈번호-기능명` (예: `feat/102-chat-ui-implementation`)
- 모든 PR(Pull Request)은 최소 1명 이상의 피어 리뷰어와 **Gemini AI 리뷰어**의 승인을 얻어야 `main`에 머지할 수 있습니다.

### 1.2 기술 스택 표준
- **Frontend**: Next.js 14 (App Router), TypeScript, Tailwind CSS, shadcn/ui
- **Backend**: Supabase (PostgreSQL, Edge Functions, Auth)
- **AI**: Vercel AI SDK, OpenAI (GPT-4o-mini), pgvector

---

## 2. ⏰ 근무 제도 및 복지

### 2.1 자율 출퇴근 및 코어 타임
- **코어 타임(Core Time)**: 오후 1시 ~ 오후 5시 (이 시간 동안은 협업을 위해 반드시 업무 가능 상태여야 합니다.)
- **리모트 근무**: 주 2회(화, 목) 자율 리모트 근무를 권장합니다.

### 2.2 복지 혜택
- **장비 지원**: 최신형 맥북 프로 및 4K 모니터 풀 세트 지원
- **도서 및 세미나**: 직무 관련 도서 구입비 및 컨퍼런스 참가비 무제한 지원
- **식대**: 점심 식대(1.5만원) 및 저녁 야근 식대 제공

---

## 3. 🔒 보안 정책 (Security First)

### 3.1 환경 변수 관리
- `.env` 파일은 절대 Git에 커밋하지 않습니다. 비밀키(API Key, DB PW)는 GitHub Secrets 또는 Supabase Vault에서 관리합니다.
- 소스 코드 내에 하드코딩된 시크릿이 발견될 경우 즉시 키를 무효화하고 보안 담당자에게 보고합니다.

### 3.2 인프라 접근 권한
- 프로덕션 DB 접근은 지정된 관리자만 가능하며, 모든 쿼리 로그는 감사(Audit) 대상입니다.
- 외부망 접속 시에는 반드시 회사 전용 VPN을 사용해야 합니다.

---

## 4. 🤝 협업 및 커뮤니케이션

### 4.1 회의 문화
- **30분 원칙**: 모든 회의는 기본 30분을 넘기지 않으며, 반드시 사전에 아젠다를 공유합니다.
- **회의록 자동화**: 모든 회의는 Pod D의 음성 인식 기능을 통해 텍스트로 기록되고 지식 맵에 보관됩니다.

### 4.2 슬랙(Slack) 활용
- 채널 용도별 분리: `#announcement`(전사 공지), `#dev-backend`(백엔드 논의), `#random`(자유 대화)
- 업무 외 시간에는 '방해 금지 모드' 설정을 권장합니다.
