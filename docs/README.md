# Docs Structure

## Folder Guide
- `api/`: API 명세 및 내부 연동 계약
- `architecture/`: DB, 프로젝트 구조, 기술 스택
- `assets/`: 디자인/참조 자산
- `guides/`: 협업 및 내부 운영 가이드
- `integrations/`: 외부/Pod 간 연동 전략 문서
- `meetings/`: 회의록
- `operations/`: 운영 로그와 의존성 기록
- `pods/`: Pod별 스킬 및 작업 문서
- `specs/`: 기능별 상세 명세
- `testing/`: 테스트 케이스와 결과
